## 進行メモ

### 導入（5分）
- 「今日は、皆さんに AI と一緒にアプリを作る体験をしてもらいます」
- 「プログラミング経験は問いません。AI に指示を出すのは人間の仕事です」

### 座学1（20分）: デモが最重要
- **最初の5分で講師がデモ**: 「チャットアプリを作って」→ AI が実装 → 画面表示
- これで受講者の目の色が変わる
- 残り15分でスライド（AI の能力と限界、指示の出し方）

### 実習1（30分）: 巡回のポイント
- 最初の5分で全員が Codespaces + Claude Code を起動できているか確認
- 「まず動かす」が最優先。完璧を求めない
- 詰まっている人がいたら「AI にエラーメッセージを貼ってみて」と促す
- 早く終わった人には「機能追加してみて」と声がけ

### 座学2（10分）: 実習1の体験を活かす
- 「さっき困ったこと、ありましたか？」→ 2-3名に聞く
- 「指示が曖昧だと AI は自分の判断で作る。それが良いとは限らない」
- 「設計を先に考えてから指示する方がうまくいく」→ 実習2へ接続

### 実習2（50分）: 設計→実装サイクル
- **最初の10分（設計フェーズ）が鍵**: 「いきなり画面を作るな。まずデータ設計」
- 受講者が設計をスキップして画面作成に行きがちなので、制止する
- 「設計を見せてもらっていいですか？」と巡回

### 実習3（20分）: 自由に
- ここは自由。早く終わった人も、まだ実習2の人も、各自のペースで

### 振り返り（15分）
- 3-5名に画面共有で発表（1人2-3分）
- 「AI に指示するとき、何が難しかったですか？」
- まとめ: 「AI はコードを書ける。でも何を作るべきかは人間が決める」

## よくあるトラブルと対処

| トラブル | 対処 |
|---|---|
| Codespaces が起動しない | ブラウザのキャッシュクリア、別ブラウザで試す |
| Claude Code が接続できない | `echo $ANTHROPIC_API_KEY` で環境変数確認 |
| npm run dev でエラー | `rm -rf node_modules && npm install` |
| AI の応答が遅い | API 負荷集中。少し待つ。Sonnet に切り替え |
| 30人同時で API 制限 | Usage Limit を上げる、またはモデルを Haiku に |

---

## API 費用の管理

- Anthropic Console → Usage → 当日のリアルタイム使用量を監視
- 目安: 30人 × 3時間 で Sonnet なら $50〜150
- Limit 設定: $200 にしておけば安全
- セミナー終了後、API Key を即削除

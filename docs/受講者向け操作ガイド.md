# AI開発セミナー 操作ガイド

このガイドでは、セミナーで使う開発環境の操作方法を説明します。
プログラミングの経験がなくても大丈夫です。順番に進めてください。

---

## 1. 開発環境を開く

### はじめて開くとき

1. Chrome で以下のページを開きます

   https://github.com/SoftAdvance/ai-dev-seminar

2. ページ上部の緑色の **「Code」** ボタンをクリックします

3. **「Codespaces」** タブをクリックします

4. **「Create codespace on master」** をクリックします

5. 画面が切り替わり、読み込みが始まります（1〜2分かかります）

6. 下の画像のような画面が表示されたら準備完了です

   ```
   ┌──────────────────────────────────────────────┐
   │ [ファイル一覧]  │  [ファイルの中身]            │
   │                 │                              │
   │  exercises/     │  （ここにプログラムが表示）   │
   │  src/           │                              │
   │  README.md      │                              │
   │                 ├──────────────────────────────│
   │                 │  [ターミナル（黒い画面）]     │
   │                 │  $                           │
   └──────────────────────────────────────────────┘
   ```

   これが皆さんの開発環境です。
   ブラウザの中で動いているので、PC には何もインストールされていません。

---

## 2. アプリを動かしてみる

1. 画面下部の黒い画面（ターミナル）をクリックします

2. 以下の文字を入力して Enter キーを押します

   ```
   npm run dev
   ```

3. 「ポートを開きますか？」のような通知が出たら **「ブラウザで開く」** をクリックします

4. 新しいタブでアプリの画面が開きます

   「AI開発セミナーへようこそ」と表示されていれば成功です！

5. このタブは閉じないでください。作業中に何度も確認で使います。

---

## 3. AI に指示を出す

### サイドバーから使う場合

1. 画面左側のアイコンの中から **Claude のアイコン**（丸いマーク）をクリックします

2. チャット画面が開きます

3. 下の入力欄に、日本語で指示を入力します。例：

   > App.tsx を書き換えて、ToDoリストアプリを作ってください

4. Enter で送信すると、AI がプログラムを書き始めます

5. AI が「ファイルを変更しますか？」と聞いてきたら **「Accept」** をクリックします

### ターミナルから使う場合

1. 黒い画面（ターミナル）で以下を入力して Enter を押します

   ```
   claude
   ```

2. AI との対話が始まります。日本語で指示を入力してください

---

## 4. 変更をブラウザで確認する

1. アプリを開いているタブ（手順2で開いたタブ）に切り替えます

2. 画面が自動的に更新されます
   （自動で変わらない場合は、ページを再読み込みしてください）

3. AI に指示した変更が反映されているか確認します

4. うまくいっていない場合は、AI に修正を依頼します。例：

   > ボタンの色を青にしてください

   > 削除ボタンが動きません。修正してください

   > エラーが出ています。（エラーの内容をそのまま貼り付ける）

---

## 5. うまくいかないときは

### 画面が真っ白になったら

1. ターミナル（黒い画面）を確認します
2. 赤い文字でエラーが表示されていたら、その内容をそのまま AI に伝えます

   > このエラーが出ています。直してください。
   > （エラーメッセージをコピーして貼り付け）

### AI の回答がおかしいと思ったら

遠慮せずに言い直してください。例：

   > そうではなくて、一覧の画面に戻るボタンがほしいです

   > もっとシンプルにしてください

### ターミナルが反応しなくなったら

**Ctrl + C** を押すと、実行中の処理を止められます。

---

## 6. 休憩するとき（ブラウザを閉じる）

作業内容は自動的に保存されています。そのままブラウザを閉じて大丈夫です。

1. そのままブラウザのタブを閉じます（× ボタン）

   ※ 特別な保存操作は不要です。

---

## 7. 作業に戻るとき

1. Chrome で以下のページを開きます

   https://github.com/codespaces

2. 自分の開発環境が一覧に表示されます

3. 環境の名前をクリックします

4. ブラウザで開発画面が再び開きます

5. ターミナル（黒い画面）で以下を入力して Enter を押すと、アプリが再起動します

   ```
   npm run dev
   ```

6. 休憩前の続きから作業できます

---

## 8. セミナー終了後

セミナーが終わったら、開発環境を削除します（PC には何も残っていないので安心してください）。

1. https://github.com/codespaces を開きます

2. 自分の開発環境の右にある **「...」**（点が3つのボタン）をクリックします

3. **「Delete」** をクリックします

4. これで完了です。
   GitHub のアカウントはそのまま残りますので、今後も活用できます。
